withRequestsMadeViaParent

HttpFeature<HttpFeatureKind.RequestsMadeViaParent>

配置当前的 HttpClient 实例，使其通过父注入器的 HttpClient 而不是直接发起请求。

默认情况下，provideHttpClient 将其注入器中的 HttpClient 配置为独立实例。例如，即使 HttpClient 在父注入器中配置了一个或多个拦截器，它们也不会拦截通过此实例发出的请求。

启用此选项后，一旦请求通过当前注入器的拦截器，它将被委托给父注入器的 HttpClient 链，而不是直接调度，并且父配置中的拦截器将应用于该请求。

如果在注入器层级结构中有多个 HttpClient 实例，则可以在多个级别使用 withRequestsMadeViaParent，这将导致请求“冒泡”上升，直到到达根级别或未配置此选项的 HttpClient。
