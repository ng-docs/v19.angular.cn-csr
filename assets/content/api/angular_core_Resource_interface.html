Resource

实验性的
代码

Resource 是一种异步依赖项（例如，API 调用的结果），它通过信号进行管理和传递。

API

  
    interface Resource<T> {}

value

Signal<T>

Resource 的当前值，如果没有当前值，则为 undefined。

status

Signal<ResourceStatus>

Resource 的当前状态，它描述了资源当前正在做什么以及对其 value 的预期。

error

Signal<unknown>

当处于 error 状态时，这将返回来自 Resource 的最后一个已知错误。

isLoading

Signal<boolean>

此资源是否正在加载新值（或重新加载现有值）。

hasValue

boolean

此资源是否具有有效的当前值。

此函数是响应式的。

@returnsboolean

reload

boolean

指示资源重新加载它可能具有的任何异步依赖项。

请注意，在发出实际后端请求之前，资源不会进入其重新加载状态。

@returnsboolean

描述

Resource 是一种异步依赖项（例如，API 调用的结果），它通过信号进行管理和传递。

创建 Resource 的常用方法是通过 resource 函数，但各种其他 API 可能会提供 Resource 实例来描述它们自己的概念。
