body

T | null

请求体；如果未设置，则为 null。

请求体并非强制不可变的，因为它们可以包含对任何用户定义数据类型的引用。但是，拦截器应注意通过将它们视为不可变来保持幂等性。

headers

HttpHeaders

此请求的传出标头。

context

HttpContext

拦截器可以使用的共享且可变上下文

reportProgress

boolean

是否应以公开进度事件的方式发出此请求。

进度事件开销很大（变更检测会在每个事件上运行），因此只有当使用者打算监视它们时才应请求它们。

注意：FetchBackend 不支持上传的进度报告。

withCredentials

boolean

是否应随传出凭据（Cookie）一起发送此请求。

responseType

"arraybuffer" | "blob" | "text" | "json"

服务器的预期响应类型。

这用于在将响应返回给请求者之前对其进行适当的解析。

method

string

传出的 HTTP 请求方法。

参数

HttpParams

发出的 URL 参数。

要在 URL 查询字符串格式中传递 HTTP 参数的字符串表示形式，可以使用 HttpParamsOptions 的 fromString。例如：

          
new HttpParams({fromString: 'angular=awesome'})

urlWithParams

string

设置了所有 URL 参数的传出 URL。

描述

一个发出的 HTTP 请求，带有可选的类型化主体。

HttpRequest 表示一个发出的请求，包括 URL、方法、标头、主体和其他请求配置选项。实例应被假定为不可变的。要修改 HttpRequest，应使用 clone 方法。
