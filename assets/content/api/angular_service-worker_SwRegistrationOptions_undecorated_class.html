SwRegistrationOptions

类
代码

令牌，可用于为 ServiceWorkerModule 提供选项，而无需在 ServiceWorkerModule.register() 中提供。

API

  
    abstract class SwRegistrationOptions {}

enabled

boolean | undefined

ServiceWorker 是否将被注册，以及相关的服务（例如 SwPush 和 SwUpdate）是否会尝试与之通信和交互。

默认值：true

scope

string | undefined

一个 URL，用于定义 ServiceWorker 的注册作用域；即，它可以控制的 URL 范围。它将在调用 ServiceWorkerContainer#register() 时使用。

registrationStrategy

string | (() => Observable<unknown>) | undefined

定义 ServiceWorker 注册策略，该策略决定了它何时在浏览器中注册。

默认行为是在应用稳定后（即，一旦没有待处理的微任务和宏任务）立即注册，其目的是尽快注册 ServiceWorker，但又不影响应用的首次加载。

不过，在某些情况下，你可能希望更好地控制 ServiceWorker 的注册时间（例如，可能存在长时间运行的超时或轮询间隔，从而阻止应用稳定）。可用的选项包括：

  • registerWhenStable:<timeout>：只要应用稳定（没有待处理的微任务/宏任务）就立即注册，但不得晚于 <timeout> 毫秒。如果应用在 <timeout> 毫秒后仍未稳定（例如，由于周期性的异步任务），ServiceWorker 仍将被注册。 如果省略 <timeout>，则仅在应用稳定后才注册 ServiceWorker。
  • registerImmediately：立即注册。
  • registerWithDelay:<timeout>：延迟 <timeout> 毫秒后注册。例如，使用 registerWithDelay:5000 在 5 秒后注册 ServiceWorker。如果省略 <timeout>，则默认为 0，这将尽快注册 ServiceWorker，但仍然是异步的，一旦所有待处理的微任务完成。
  • 一个 Observable 工厂函数：一个返回 Observable 的函数。 该函数将在运行时用于获取和订阅 Observable，并且 ServiceWorker 将在发出第一个值后立即注册。

默认值：'registerWhenStable:30000'

说明

令牌，可用于为 ServiceWorkerModule 提供选项，而无需在 ServiceWorkerModule.register() 中提供。
