AfterRenderPhase

枚举

代码

在此阶段运行 afterRender 或 afterNextRender 回调。

API

  
    enum AfterRenderPhase {}

EarlyRead

对于只需要在后续的 AfterRenderPhase.Write 回调之前从 DOM 中读取的回调，请使用 AfterRenderPhase.EarlyRead，例如，执行浏览器本身不支持的自定义布局。如果读取可以等到写入阶段之后，则首选 AfterRenderPhase.Read 阶段。绝不要在此阶段写入 DOM。

使用此值可能会降低性能。 相反，如果可能，请首选使用内置浏览器功能。

写入

对于仅写入 DOM 的回调，请使用 AfterRenderPhase.Write。绝不要在此阶段从 DOM 中读取。

MixedReadWrite

对于从 DOM 读取或写入 DOM，但尚未重构为使用不同阶段的回调，请使用 AfterRenderPhase.MixedReadWrite。如果可以将工作分配到其他阶段，则绝不要使用此阶段。

使用此值会显著降低性能。 相反，请首选将工作分配到适当的阶段回调中。

读取

对于仅从 DOM 中读取的回调，请使用 AfterRenderPhase.Read。绝不要在此阶段写入 DOM。

描述

在此阶段运行 afterRender 或 afterNextRender 回调。

同一阶段中的回调按照注册顺序运行。各阶段在每次渲染后按以下顺序运行：

  1. AfterRenderPhase.EarlyRead
  2. AfterRenderPhase.Write
  3. AfterRenderPhase.MixedReadWrite
  4. AfterRenderPhase.Read

Angular 无法验证或强制执行阶段是否被正确使用，而是依赖于每位开发者来遵循为每个值记录的指南，并仔细选择合适的阶段，并在必要时重构其代码。通过这样做，Angular 能够更好地最大限度地减少与手动 DOM 访问相关的性能下降，从而确保为你的应用或库的最终用户提供最佳体验。
