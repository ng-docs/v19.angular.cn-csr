Injectable

decorator
代码

此装饰器用于将某个类标记为可被提供和作为依赖项注入。

服务和依赖注入 (DI) 简介

API

  
    @Injectable ({})

providedIn

Type<any> | "root" | "platform" | "any" | null | undefined

确定哪个注入器将提供此 Injectable。

  • Type<any> - 将此 Injectable 与 @NgModule 或其他 InjectorType 关联起来。此选项已弃用。
  • 'null'：等效于 undefined。此 Injectable 不会自动在任何作用域中提供，必须将其添加到 @NgModule、@Component 或 @Directive 的 providers 数组中。

以下选项指定了应在以下注入器之一中提供此 Injectable：

  • 'root'：大多数应用中的应用级注入器。
  • 'platform'：一个特殊的单例平台注入器，由页面上的所有应用共享。
  • 'any'：在每个惰性加载模块中提供一个唯一的实例，而所有急性加载模块共享一个实例。此选项已弃用。

使用须知

使用 @Injectable 标记类可确保编译器将生成必要的元数据，以便在该类被注入时创建该类的依赖项。

以下范例展示了如何正确标记服务类，以便在创建时可以注入支持服务。

          
@Injectable()        class UsefulService {}        @Injectable()        class NeedsService {          constructor(public service: UsefulService) {}        }        const injector = Injector.create({          providers: [            {provide: NeedsService, deps: [UsefulService]},            {provide: UsefulService, deps: []},          ],        });        expect(injector.get(NeedsService).service instanceof UsefulService).toBe(true);
