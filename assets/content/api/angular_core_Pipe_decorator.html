风格指南：管道名称

API

  
    @Pipe ({})

name

string

在模板绑定中使用的管道名称。 通常使用 lowerCamelCase 格式， 因为名称不能包含连字符。

pure

boolean | undefined

当为 true 时，管道是纯管道，这意味着仅当其输入参数发生变化时，才会调用 transform() 方法。管道默认是纯管道。

如果管道具有内部状态（即，结果取决于除其参数之外的状态），请将 pure 设置为 false。 在这种情况下，即使参数没有更改，也会在每个变更检测周期调用该管道。

standalone

boolean | undefined

标记为 standalone 的 Angular 管道不需要在 NgModule 中声明。此类管道不依赖 NgModule 的任何“中间上下文”（例如，配置的提供者）。

有关独立组件、指令和管道的更多信息，请参阅本指南。

描述

将类标记为管道并提供配置元数据的装饰器。

管道类必须实现 PipeTransform 接口。 例如，如果名称为“myPipe”，请使用如下模板绑定表达式：

          
{{ exp | myPipe }}

表达式的结果会传递给管道的 transform() 方法。

管道必须属于 NgModule 才能在模板中使用。要使其成为 NgModule 的成员，请在 NgModule 元数据的 declarations 字段中列出它。
