provideExperimentalCheckNoChangesForDebug

EnvironmentProviders

用于定期验证在表达式检查过后是否发生了更改。

@paramoptions{ interval?: number | undefined; useNgZoneOnStable?: boolean | undefined; exhaustive?: boolean | undefined; }

用于配置何时执行检查。

  • interval 将定期在应用视图上运行详尽的 checkNoChanges
  • useNgZoneOnStable 将使用 ZoneJS 来确定何时可能已在应用中使用 ZoneJS 驱动变更检测的情况下运行变更检测。当 NgZone.onStable 将要发出时，会检查附着到 ApplicationRef 的所有视图是否有更改。
  • “exhaustive”表示将检查附着到 ApplicationRef 的所有视图以及这些视图的所有后代是否有更改（不包括通过 ChangeDetectorRef.detach() 分离的那些子树）。这很有用，因为在常规变更检测之后运行的检查不适用于使用 ChangeDetectionStrategy.OnPush 的组件。此检查将显示 OnPush 组件隐藏的任何现有错误。默认情况下，此检查是详尽的，并将始终检查所有视图，而不管其“脏”状态和 ChangeDetectionStrategy 如何。

当 useNgZoneOnStable 选项为 true 时，此函数将提供其自己的 NgZone 实现，并且需要在任何其他 NgZone 提供者（包括 provideZoneChangeDetection() 和 provideExperimentalZonelessChangeDetection()）之后引入。

@returnsEnvironmentProviders
