说明

获取 Observable 的当前值，作为响应式的 Signal。

toSignal 返回一个 Signal，它通过订阅给定的 Observable，来提供对该 Observable 所产生值的同步响应式访问。返回的 Signal 将始终具有订阅发出的最新值，并且如果 Observable 发生错误，则会抛出错误。

当 requireSync 设置为 true 时，toSignal 将断言 Observable 在订阅后立即生成一个值。在这种情况下，不需要 initialValue，并且返回的 signal 不包含 undefined 类型。

默认情况下，当当前的注入上下文被销毁时，订阅将自动清理。例如，当在组件的构造函数中调用 toSignal 时，订阅将在组件销毁时清理。如果注入上下文不可用，则可以改为传入显式的 Injector。

如果订阅应持续到 Observable 本身完成，则可以改为指定 manualCleanup 选项，这将禁用自动订阅拆除。在这种配置中，也不需要注入上下文。
