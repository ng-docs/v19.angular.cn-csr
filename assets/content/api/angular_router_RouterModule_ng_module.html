创建一个模块，其中包含所有路由器指令和一个注册路由的提供者， 而无需创建新的 Router 服务。 当为子模块和惰性加载的子模块注册时，按如下方式创建 NgModule：

描述

为应用中定义的视图之间的应用内导航添加指令和提供者。 使用 Angular 的 Router 服务以声明方式指定应用状态并管理状态转换。

你可以多次导入此 NgModule，每个惰性加载的捆绑包导入一次。 但是，只能有一个 Router 服务处于活动状态。 为了确保这一点，导入此模块时有两种注册路由的方法：

  • forRoot() 方法创建一个 NgModule，其中包含所有指令、给定的路由以及 Router 服务本身。
  • forChild() 方法创建一个 NgModule，其中包含所有指令和给定的路由，但不包括 Router 服务。
