@angular/core

NgModule

decorator
代码

用来标记某个类为 NgModule 并提供配置元数据的装饰器。

API

  
    @NgModule ({})

providers

(Provider | EnvironmentProviders)[] | undefined

本模块的注入器中可用的可注入对象集。

declarations

(any[] | Type<any>)[] | undefined

属于本模块的组件、指令和管道（可声明对象）的集合。

imports

(any[] | Type<any> | ModuleWithProviders<{}>)[] | undefined

本模块中的模板可用的那些导出的可声明对象的 NgModule 集合。

exports

(any[] | Type<any>)[] | undefined

本 NgModule 中声明的组件、指令和管道的集合，它们可以在导入此 NgModule 的任何 NgModule 所属的组件的模板中使用。导出的声明是该模块的公共 API。

一个可声明对象只属于一个 NgModule。一个模块可以在其导出列表中列出另一个模块，在这种情况下，该模块的所有公共声明都会被导出。

bootstrap

(any[] | Type<any>)[] | undefined

当此模块启动时要引导启动的组件集合。

schemas

(any[] | SchemaMetadata)[] | undefined

用于声明 NgModule 中允许使用的元素的架构集合。既不是 Angular 组件也不是指令的元素和属性必须在架构中声明。

允许的值为 NO_ERRORS_SCHEMA 和 CUSTOM_ELEMENTS_SCHEMA。

id

string | undefined

在 getNgModuleById 中唯一标识此 NgModule 的名称或路径。 如果保留为 undefined，则 NgModule 不会注册到 getNgModuleById 中。

jit

true | undefined

如果存在，预先编译器（AOT）将忽略此模块。它会保留在分发后的代码中，即时编译器（JIT）会尝试在运行时在浏览器中对其进行编译。为了确保行为正确，应用必须导入 @angular/compiler。
