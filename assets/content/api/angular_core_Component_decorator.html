@angular/core

组件

decorator
代码

用来将类标记为 Angular 组件的装饰器，并提供配置元数据，以确定组件在运行时应如何处理、实例化和使用。

API

  
    @Component ({})

changeDetection

ChangeDetectionStrategy | undefined

要用于此组件的变更检测策略。

当组件被实例化时，Angular 会创建一个变更检测器，它负责传播组件的绑定。策略之一是：

  • ChangeDetectionStrategy#OnPush 将策略设置为 CheckOnce（按需检测）。
  • ChangeDetectionStrategy#Default 将策略设置为 CheckAlways。

viewProviders

Provider[] | undefined

定义了对其视图 DOM 子级可见的可注入对象集合。请参阅范例。

moduleId

string | undefined
@deprecated

此选项没有任何效果。将在 Angular v17 中移除。

包含该组件的模块的模块 ID。组件必须能够求解模板和样式的相对 URL。SystemJS 在每个模块中公开 __moduleName 变量。在 CommonJS 中，可以将其设置为 module.id。

templateUrl

string | undefined

Angular 组件的模板文件的相对路径或绝对 URL。如果已提供，请勿使用 template 提供内联模板。

template

string | undefined

Angular 组件的内联模板。如果已提供，请勿使用 templateUrl 提供模板文件。

styleUrl

string | undefined

用于此组件的包含 CSS 样式表的文件的相对路径或绝对 URL。

styleUrls

string[] | undefined

用于此组件的包含 CSS 样式表的文件的相对路径或绝对 URL。

styles

string | string[] | undefined

在此组件中使用的一个或多个内联 CSS 样式表。

animations

any[] | undefined

一个或多个动画 trigger() 调用，其中包含 state() 和 transition() 定义。 请参阅动画指南和动画 API 文档。

encapsulation

ViewEncapsulation | undefined

组件样式的封装策略。 可能的值：

  • ViewEncapsulation.Emulated：应用修改后的组件样式，以模拟原生的 Shadow DOM CSS 封装行为。
  • ViewEncapsulation.None：全局应用组件样式，没有任何形式的封装。
  • ViewEncapsulation.ShadowDom：使用浏览器的原生 Shadow DOM API 来封装样式。

如果未提供，则该值取自 CompilerOptions，后者默认为 ViewEncapsulation.Emulated。

如果策略是 ViewEncapsulation.Emulated 并且组件没有 styles 或 styleUrls，则策略会自动切换为 ViewEncapsulation.None。

interpolation

[string, string] | undefined
@deprecated

请改为使用 Angular 的默认插值分隔符。

覆盖默认的插值开始和结束分隔符（{{ 和 }}）。

preserveWhitespaces

boolean | undefined

true 表示保留，false 表示从已编译的模板中删除可能多余的空格字符。 空格字符是那些与 JavaScript 正则表达式中的 \s 字符类匹配的字符。 默认为 false，除非在编译器选项中被覆盖。

standalone

boolean | undefined

标记为 standalone 的 Angular 组件不需要在 NgModule 中声明。 这样的组件通过 imports 属性直接管理它们自己的模板依赖项（在模板中使用的组件、指令和管道）。

关于独立组件、指令和管道的更多信息，请参见本指南。

imports

(readonly any[] | Type<any>)[] | undefined

imports 属性指定了独立组件的模板依赖项——即可以在其模板中使用的那些指令、组件和管道。独立组件可以导入其他独立组件、指令和管道，以及现有的 NgModule。

此属性（Property）仅适用于独立组件 - 为 NgModule 中声明的组件指定它会生成编译错误。

关于独立组件、指令和管道的更多信息，请参见本指南。

schemas

SchemaMetadata[] | undefined

用于声明允许在独立组件中使用的元素的架构集合。 既不是 Angular 组件也不是指令的元素和属性（Property）必须在架构中声明。

此属性（Property）仅适用于独立组件 - 为 NgModule 中声明的组件指定它会生成编译错误。

关于独立组件、指令和管道的更多信息，请参见本指南。

selector

string | undefined

CSS 选择器，用于在模板中标识此指令并触发指令的实例化。

声明为以下之一：

  • element-name：按元素名称选择。
  • .class：按类名选择。
  • [attribute]：按属性（Attribute）名选择。
  • [attribute=value]：按属性（Attribute）名和值选择。
  • :not(sub_selector)：仅当元素与 sub_selector 不匹配时才选择。
  • selector1, selector2：如果 selector1 或 selector2 匹配，则选择。

Angular 只允许指令应用于不跨越元素边界的 CSS 选择器。

对于以下模板 HTML，带有 input[type=text] 选择器的指令将仅在 <input type="text"> 元素上实例化。

          
<form>  <input type="text">  <input type="radio"><form>

inputs

(string | { name: string; alias?: string | undefined; required?: boolean | undefined; transform?: ((value: any) => any) | undefined; })[] | undefined

枚举指令的数据绑定输入属性（Property）集

在变更检测期间，Angular 会自动更新输入属性（Property）。inputs 属性接受字符串或对象字面量，它们会配置应该作为输入公开的指令属性（property）。

当传入对象字面量时，name 属性会指明输入应该写入到类上的哪个属性（property），而 alias 则决定了在模板绑定中可用的输入名称。required 属性表示此输入是必要的，如果在使用该指令时未传入此输入，则会触发编译时错误。

当将字符串传入 inputs 数组时，它可以具有 'name' 或 'name: alias' 的格式，其中 name 是指令应写入的类上的属性（property），而 alias 确定了在模板绑定中可用的输入名称。基于字符串的输入定义被假定为是可选的。

outputs

string[] | undefined

枚举事件绑定的输出属性（Property）的集合。

当输出属性（Property）发出事件时，将调用模板中附着到该事件的事件处理器。

outputs 属性定义了一组从 directiveProperty 到 alias 的配置：

  • directiveProperty 指定了发出事件的组件属性（property）。
  • alias 指定了事件处理器附着到的 DOM 属性（Property）。

providers

Provider[] | undefined

使用一个令牌配置此指令或组件的注入器，该令牌映射到依赖项的提供者。

exportAs

string | undefined

定义可在模板中用于将此指令赋值给变量的名称。

queries

{ [key: string]: any; } | undefined

配置将注入到指令中的查询。

内容查询在 ngAfterContentInit 回调被调用之前设置。视图查询在 ngAfterViewInit 回调被调用之前设置。

host

{ [key: string]: string; } | undefined

使用一组键值对，将类属性（property）映射到宿主元素绑定，以用于属性（property）、Attribute 和事件。

在变更检测期间，Angular 会自动检查宿主属性（Property）绑定。如果绑定发生更改，Angular 会更新指令的宿主元素。

当键是宿主元素的属性（property）时，该属性值将传播到指定的 DOM 属性（Property）。

当键是 DOM 中的静态 Attribute 时，该 Attribute 值将传播到宿主元素中的指定属性（property）。

对于事件处理：

  • 键是指令监听的 DOM 事件。要监听全局事件，请将目标添加到事件名称中。目标可以是 window、document 或 body。
  • 该值是事件发生时要执行的语句。如果语句求值结果为 false，则会对 DOM 事件应用 preventDefault。处理函数方法可以引用 $event 局部变量。

jit

true | undefined

如果存在，则 AOT 编译器会忽略此指令/组件。它会保留在分发代码中，而 JiT 编译器会尝试在运行时在浏览器中对其进行编译。为了确保行为正确，应用必须导入 @angular/compiler。

hostDirectives

(Type<unknown> | { directive: Type<unknown>; inputs?: string[] | undefined; outputs?: string[] | undefined; })[] | undefined

每当指令匹配时，都应应用于宿主的独立指令。默认情况下，宿主指令的任何输入或输出在宿主上都不可用，除非它们在 inputs 或 outputs 属性（Property）中指定。

你还可以通过在原始输入或输出名称后加上冒号和别名来为输入和输出设置别名。例如，如果通过 hostDirectives 应用的指令定义了一个名为 menuDisabled 的输入，你可以通过将 'menuDisabled: disabled' 作为条目添加到 inputs 中，将其别名为 disabled。

描述

用来将类标记为 Angular 组件的装饰器，并提供配置元数据，以确定组件在运行时应如何处理、实例化和使用。

组件是 Angular 应用最基本的 UI 构建块。一个 Angular 应用包含一棵 Angular 组件树。

Angular 组件是指令的一个子集，始终与模板关联。与其他指令不同，对于模板中的给定元素，只能实例化一个组件。

独立组件可以直接导入到任何其他独立组件或 NgModule 中。另一方面，基于 NgModule 的应用要求组件属于 NgModule，以便它们可用于另一个组件或应用。要使组件成为 NgModule 的成员，请在 NgModule 元数据的 declarations 字段中列出它。

请注意，除了这些用于配置指令的选项之外，你还可以通过实现生命周期钩子来控制组件的运行时行为。有关更多信息，请参阅生命周期钩子指南。

使用须知

设置组件输入

以下范例创建了一个具有两个数据绑定属性（property）的组件，这些属性由 inputs 值指定。

          
@Component({  selector: 'app-bank-account',  inputs: ['bankName', 'id: account-id'],  template: ` Bank Name: {{ bankName }} Account Id: {{ id }} `,  standalone: false,})export class BankAccountComponent {  bankName: string | null = null;  id: string | null = null;  // this property is not bound, and won't be automatically updated by Angular  normalizedBankName: string | null = null;}@Component({  selector: 'app-my-input',  template: ` <app-bank-account bankName="RBC" account-id="4747"> </app-bank-account>`,  standalone: false,})export class MyInputComponent {}

设置组件输出

以下范例显示了两个事件发射器，它们按时间间隔发射事件。一个每秒发射一次输出，而另一个每五秒发射一次。

          
@Directive({  selector: 'app-interval-dir',  outputs: ['everySecond', 'fiveSecs: everyFiveSeconds'],  standalone: false,})export class IntervalDirComponent {  everySecond = new EventEmitter<string>();  fiveSecs = new EventEmitter<string>();  constructor() {    setInterval(() => this.everySecond.emit('event'), 1000);    setInterval(() => this.fiveSecs.emit('event'), 5000);  }}@Component({  selector: 'app-my-output',  template: `    <app-interval-dir (everySecond)="onEverySecond()" (everyFiveSeconds)="onEveryFiveSeconds()">    </app-interval-dir>  `,  standalone: false,})export class MyOutputComponent {  onEverySecond() {    console.log('second');  }  onEveryFiveSeconds() {    console.log('five seconds');  }}

使用视图提供者注入类

以下简单范例使用组件元数据中指定的视图提供者将一个类注入到组件中：

          
class Greeter {   greet(name:string) {     return 'Hello ' + name + '!';   }}@Directive({  selector: 'needs-greeter'})class NeedsGreeter {  greeter:Greeter;  constructor(greeter:Greeter) {    this.greeter = greeter;  }}@Component({  selector: 'greet',  viewProviders: [    Greeter  ],  template: `<needs-greeter></needs-greeter>`})class HelloWorld {}

保留空白符

删除空白符可以大大减小预先编译（AoT）生成的代码大小并加快视图创建速度。从 Angular 6 开始，preserveWhitespaces 的默认值为 false（删除空白符）。要更改应用中所有组件的默认设置，请设置预先编译器（AoT）的 preserveWhitespaces 选项。

默认情况下，预先编译器（AoT）按如下方式删除空白字符：

  • 修剪模板开头和结尾的所有空白符。
  • 删除仅包含空白符的文本节点。例如，
          
<button>Action 1</button>  <button>Action 2</button>

变为：

          
<button>Action 1</button><button>Action 2</button>
  • 将文本节点中的一系列空白字符替换为单个空格。例如，<span>\n some text\n</span> 变为 <span> some text </span>。
  • 不会更改 HTML 标签（如 <pre> 或 <textarea>）内的文本节点，在这些标签中，空白字符是有意义的。

请注意，这些转换可能会影响 DOM 节点布局，尽管影响应该是最小的。

你可以重写默认行为以在模板的某些片段中保留空白字符。例如，你可以使用 ngPreserveWhitespaces 属性排除整个 DOM 子树：

          
<div ngPreserveWhitespaces>    whitespaces are preserved here    <span>    and here </span></div>

你可以使用 &ngsp; 强制在文本节点中保留单个空格，Angular 的模板编译器会将其替换为空格字符：

          
<a>Spaces</a>&ngsp;<a>between</a>&ngsp;<a>links.</a><!-- compiled to be equivalent to: <a>Spaces</a> <a>between</a> <a>links.</a>  -->

请注意，当 preserveWhitespaces 选项设置为 false 时，&ngsp; 序列仍然会折叠为一个空格字符。

          
<a>before</a>&ngsp;&ngsp;&ngsp;<a>after</a><!-- compiled to be equivalent to: <a>before</a> <a>after</a> -->

要保留空白字符序列，请使用 ngPreserveWhitespaces 属性。
