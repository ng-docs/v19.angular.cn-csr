@angular/core/rxjs-interop

ToSignalOptions

interface
代码

toSignal 的选项。

API

  
    interface ToSignalOptions<T> {}

initialValue

unknown

由 toSignal 生成的信号的初始值。

在可观察者发出它的第一个值之前，这将是信号的值。

requireSync

boolean | undefined

是否要求当 toSignal 订阅时，可观察者同步发出值。

如果为 true，则 toSignal 将断言可观察者在订阅时立即生成一个值。设置此选项消除了在信号类型中处理 undefined 或提供 initialValue 的需要，但如果未满足此要求，则会产生运行时错误。

injector

Injector | undefined

Injector，它将提供用于清理 Observable 订阅的 DestroyRef。

如果未提供，则将从当前的 注入上下文 中检索 DestroyRef，除非请求手动清理。

manualCleanup

boolean | undefined

当 toSignal 的创建上下文被销毁时，是否应自动清理订阅（通过 DestroyRef）。

如果启用了手动清理，则不使用 DestroyRef，并且订阅将持续存在，直到 Observable 本身完成。

rejectErrors

boolean | undefined

toSignal 是否应将来自 Observable 错误通道的错误抛回 RxJS，在那里它们将被视为未捕获的异常。

在实践中，这意味着由 toSignal 返回的信号将永远保持返回最后一个好值，因为发生错误的可观察者不会产生进一步的值。此选项模拟了 async 管道的行为。

equal

ValueEqualityFn<T> | undefined

一个比较函数，它定义了可观察者发出的值的相等性。

如果提供了初始值，则针对该初始值执行相等性比较。
