@angular/router

IsActiveMatchOptions

interface
代码

一组选项，用于指定在给定当前路由器状态的 UrlTree 的情况下，如何确定 UrlTree 是否处于激活状态。

Router#isActive

API

  
    interface IsActiveMatchOptions {}

matrixParams

"exact" | "subset" | "ignored"

定义了比较两个 UrlTree 的矩阵参数的策略。

矩阵参数匹配取决于匹配段的策略。也就是说，如果 paths 选项设置为 'subset'，则只会比较匹配段的矩阵参数。

  • 'exact'：要求匹配的段也必须具有精确的矩阵参数匹配。
  • 'subset'：路由器激活的 UrlTree 中的匹配段可能包含额外的矩阵参数，但有问题的 UrlTree 中存在的矩阵参数必须匹配。
  • 'ignored'：当比较 UrlTree 时，矩阵参数将被忽略。

queryParams

"exact" | "subset" | "ignored"

定义了比较两个 UrlTree 的查询参数的策略。

  • 'exact'：查询参数必须精确匹配。
  • 'subset'：激活的 UrlTree 可能包含额外的参数，但必须匹配有问题的 UrlTree 中存在的任何参数的键和值。
  • 'ignored'：当比较 UrlTree 时，查询参数将被忽略。

paths

"exact" | "subset"

定义了比较 UrlTree 的 UrlSegment 的策略。

  • 'exact'：每个 UrlTree 中的所有段都必须匹配。
  • 'subset'：如果 UrlTree 是活动路由的子树，则它将被确定为激活状态。也就是说，活动路由可能包含额外的段，但必须至少包含有问题的 UrlTree 的所有段。

fragment

"exact" | "ignored"
  • 'exact'：表明 UrlTree 片段必须相等。
  • 'ignored'：在确定 UrlTree 是否处于激活状态时，将不会比较片段。
