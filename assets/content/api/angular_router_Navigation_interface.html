Navigation

interface
代码

关于导航操作的信息。 使用 Router.getCurrentNavigation() 方法检索最新的导航对象。

API

  
    interface Navigation {}

id

number

当前导航的唯一标识符。

initialUrl

UrlTree

在导航之前，传递给 Router#navigateByUrl() 调用的目标 URL。这是路由器解析或应用重定向之前的值。

extractedUrl

UrlTree

使用 UrlHandlingStrategy.extract() 解析后的初始目标 URL。

finalUrl

UrlTree | undefined

应用重定向后提取的 URL。 此 URL 可能不会立即提供，因此此属性可以是 undefined。 保证在 RoutesRecognized 事件触发后设置。

trigger

"imperative" | "popstate" | "hashchange"

标识此导航是如何触发的。

  • “imperative”——由 router.navigateByUrl 或 router.navigate 触发。
  • “popstate”——由 popstate 事件触发。
  • “hashchange”——由 hashchange 事件触发。

extras

NavigationExtras

控制用于此导航的策略的选项。 请参阅 NavigationExtras。

previousNavigation

Navigation | null

先前成功的 Navigation 对象。只有一个先前的导航可用，因此，此先前的 Navigation 对象的 previousNavigation 属性值为 null。

说明

关于导航操作的信息。 使用 Router.getCurrentNavigation() 方法检索最新的导航对象。

  • id ：当前导航的唯一标识符。
  • initialUrl ：在导航之前，传递给 Router#navigateByUrl() 调用的目标 URL。 这是路由器解析或应用重定向之前的值。
  • extractedUrl ：使用 UrlSerializer.extract() 解析后的初始目标 URL。
  • finalUrl ：应用重定向后提取的 URL。 此 URL 可能不会立即提供，因此此属性可以是 undefined。 保证在 RoutesRecognized 事件触发后设置。
  • trigger ：标识此导航是如何触发的。 ——“imperative”——由 router.navigateByUrl 或 router.navigate 触发。 ——“popstate”——由 popstate 事件触发。 ——“hashchange”——由 hashchange 事件触发。
  • extras ：一个 NavigationExtras 选项对象，用于控制此导航所用的策略。
  • previousNavigation ：先前成功的 Navigation 对象。只有一个先前的导航可用，因此，此先前的 Navigation 对象的 previousNavigation 属性值为 null。
