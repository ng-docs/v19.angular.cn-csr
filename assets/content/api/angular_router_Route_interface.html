@angular/router

Route

interface
code

一个配置对象，用于定义单个路由。一组路由收集在 Routes 数组中，以定义 Router 配置。路由器尝试将给定 URL 的片段与每个路由进行匹配，使用此对象中定义的配置选项。

API

  
    interface Route {}

title

string | Type<Resolve<string>> | ResolveFn<string> | undefined

用于为路由定义页面标题。它可以是静态字符串，也可以是实现了 Resolve 的 Injectable。

path

string | undefined

要匹配的路径。不能与自定义的 matcher 函数一起使用。 一个使用路由器匹配语法的 URL 字符串。 可以是一个通配符 (**)，以匹配任何 URL（见下面的“用法说明”）。 默认为 "/"（根路径）。

pathMatch

"prefix" | "full" | undefined

路径匹配策略，为 'prefix' 或 'full' 之一。 默认为 'prefix'。

默认情况下，路由器从左到右检查 URL 元素，以查看 URL 是否与给定的路径匹配，并在配置匹配时停止。重要的是，URL 的每个段仍然必须有配置匹配。例如，如果路由的子路由之一与段 'user' 匹配，则 '/team/11/user' 匹配前缀 'team/:id'。也就是说，URL '/team/11/user' 匹配配置 {path: 'team/:id', children: [{path: ':user', component: User}]}，但当没有子路由时（如 {path: 'team/:id', component: Team}）则不匹配。

路径匹配策略 'full' 针对整个 URL 进行匹配。 当重定向空路径路由时，这样做很重要。 否则，由于空路径是任何 URL 的前缀，因此即使导航到重定向目标，路由器也会应用重定向，从而创建无限循环。

matcher

UrlMatcher | undefined

自定义 URL 匹配函数。不能与 path 一起使用。

component

Type<any> | undefined

当路径匹配时要实例化的组件。 如果子路由指定了组件，则可以为空。

loadComponent

(() => Type<unknown> | Observable<Type<unknown> | DefaultExport<Type<unknown>>> | Promise<Type<unknown> | DefaultExport<Type<unknown>>>) | undefined

一个指定惰性加载组件的对象。

redirectTo

string | RedirectFunction | undefined

当路径匹配时，要重定向到的 URL 或返回 URL 的函数。

如果 URL 以斜杠 (/) 开头，或者函数返回 UrlTree，则为绝对路径，否则相对于路径 URL。

RedirectFunction 在注入上下文中运行，因此它可以调用 inject 以获取任何必要的依赖项。

如果不存在，路由器不会重定向。

outlet

string | undefined

RouterOutlet 对象的名称，当路径匹配时，组件可以放置在该对象中。

canActivate

any[] | undefined

CanActivateFn 或 DI 令牌的数组，用于查找 CanActivate() 处理程序，以确定当前用户是否被允许激活组件。默认情况下，任何用户都可以激活。

当使用函数而不是 DI 令牌时，该函数可以调用 inject 以获取任何必需的依赖项。这个 inject 调用必须在同步上下文中完成。

canMatch

any[] | undefined

CanMatchFn 或 DI 令牌的数组，用于查找 CanMatch() 处理程序，以确定当前用户是否被允许匹配 Route。默认情况下，任何路由都可以匹配。

当使用函数而不是 DI 令牌时，该函数可以调用 inject 以获取任何必需的依赖项。这个 inject 调用必须在同步上下文中完成。

canActivateChild

any[] | undefined

CanActivateChildFn 或 DI 令牌的数组，用于查找 CanActivateChild() 处理程序，以确定当前用户是否被允许激活组件的子组件。默认情况下，任何用户都可以激活子组件。

当使用函数而不是 DI 令牌时，该函数可以调用 inject 以获取任何必需的依赖项。这个 inject 调用必须在同步上下文中完成。

canDeactivate

any[] | undefined

CanDeactivateFn 或 DI 令牌的数组，用于查找 CanDeactivate() 处理程序，以确定当前用户是否被允许停用组件。默认情况下，任何用户都可以停用。

当使用函数而不是 DI 令牌时，该函数可以调用 inject 以获取任何必需的依赖项。这个 inject 调用必须在同步上下文中完成。

canLoad

any[] | undefined
@deprecated

请改用 canMatch

CanLoadFn 或 DI 令牌的数组，用于查找 CanLoad() 处理程序，以确定当前用户是否被允许加载组件。默认情况下，任何用户都可以加载。

当使用函数而不是 DI 令牌时，该函数可以调用 inject 以获取任何必需的依赖项。这个 inject 调用必须在同步上下文中完成。

data

Data | undefined

通过 ActivatedRoute 提供给组件的、由开发人员定义的额外数据。默认情况下，不传递任何额外数据。

resolve

ResolveData | undefined

用于查找数据求解器的 DI 令牌的映射表。请参阅 Resolve。

children

Routes | undefined

一个 Route 子对象数组，用于指定嵌套路由配置。

loadChildren

LoadChildrenCallback | undefined

一个用于指定惰性加载的子路由的对象。

runGuardsAndResolvers

RunGuardsAndResolvers | undefined

一个策略，用于指定何时在路由上运行守卫和求解器。

当路由被激活或停用时，守卫和/或求解器将始终运行。当路由未更改时，默认行为与 paramsChange 相同。

paramsChange：当路径或路径参数更改时，重新运行守卫和求解器。这不包括查询参数。此选项为默认选项。

  • always：每次执行时都运行。
  • pathParamsChange：当路径参数更改时，重新运行守卫和求解器。这不会比较矩阵参数或查询参数。
  • paramsOrQueryParamsChange：当路径、矩阵或查询参数更改时运行。
  • pathParamsOrQueryParamsChange：当路径参数更改或查询参数已更改时，重新运行守卫和求解器。这不包括矩阵参数。

providers

(Provider | EnvironmentProviders)[] | undefined

一个 Provider 数组，用于此 Route 及其 children。

Router 将为此 Route 创建一个新的 EnvironmentInjector，并将其用于此 Route 及其 children。如果此路由还具有返回 NgModuleRef 的 loadChildren 函数，则此注入器将用作惰性加载模块的父注入器。

描述

一个配置对象，用于定义单个路由。一组路由收集在 Routes 数组中，以定义 Router 配置。路由器尝试将给定 URL 的片段与每个路由进行匹配，使用此对象中定义的配置选项。

支持静态路由、参数化路由、重定向路由和通配符路由，以及自定义路由数据和求解方法。

有关详细用法信息，请参阅路由指南。

用法说明

简单配置

以下路由指定，例如，当导航到 /team/11/user/bob 时，路由器将创建 'Team' 组件，并在其中包含 'User' 子组件。

          
[{  path: 'team/:id', component: Team,  children: [{    path: 'user/:name',    component: User  }]}]

多出口

以下路由使用多个出口创建同级组件。当导航到 /team/11(aux:chat/jim) 时，路由器将创建 'Team' 组件，并在其旁边创建 'Chat' 组件。'Chat' 组件被放置到 'aux' 出口中。

          
[{  path: 'team/:id',  component: Team}, {  path: 'chat/:user',  component: Chat  outlet: 'aux'}]

通配符

以下路由使用通配符表示法来指定一个组件，无论导航到何处，该组件始终会被实例化。

          
[{  path: '**',  component: WildcardComponent}]

重定向

以下路由使用 redirectTo 属性在查找子路径时忽略给定 URL 的片段。

当导航到 '/team/11/legacy/user/jim' 时，路由器将 URL 片段 '/team/11/legacy/user/jim' 更改为 '/team/11/user/jim'，然后实例化 Team 组件，并在其中包含 User 子组件。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: 'legacy/user/:name',    redirectTo: 'user/:name'  }, {    path: 'user/:name',    component: User  }]}]

重定向路径可以是相对路径（如本例所示），也可以是绝对路径。如果我们将示例中的 redirectTo 值更改为绝对 URL 片段 '/user/:name'，则结果 URL 也是绝对路径 '/user/jim'。

空路径

空路径路由配置可用于实例化不“消耗”任何 URL 片段的组件。

在以下配置中，当导航到 /team/11 时，路由器将实例化 'AllUsers' 组件。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: AllUsers  }, {    path: 'user/:name',    component: User  }]}]

空路径路由可以有子路由。在以下示例中，当导航到 /team/11/user/jim 时，路由器将实例化包装器组件，并在其中包含用户组件。

请注意，空路径路由会继承其父路由的参数和数据。

          
[{  path: 'team/:id',  component: Team,  children: [{    path: '',    component: WrapperCmp,    children: [{      path: 'user/:name',      component: User    }]  }]}]

匹配策略

默认的路径匹配策略是 'prefix'，这意味着路由器从左侧检查 URL 元素，以查看 URL 是否与指定的路径匹配。例如，'/team/11/user' 匹配 'team/:id'。

          
[{  path: '',  pathMatch: 'prefix', //default  redirectTo: 'main'}, {  path: 'main',  component: Main}]

你可以指定路径匹配策略 'full'，以确保路径覆盖整个未消耗的 URL。在重定向空路径路由时，这样做很重要。否则，由于空路径是任何 URL 的前缀，因此即使导航到重定向目标，路由器也会应用重定向，从而创建无限循环。

在以下示例中，提供 'full' pathMatch 策略可确保路由器仅在导航到 '/' 时才应用重定向。

          
[{  path: '',  pathMatch: 'full',  redirectTo: 'main'}, {  path: 'main',  component: Main}]

无组件路由

你可以在同级组件之间共享参数。例如，假设两个同级组件应该彼此相邻，并且它们都需要一个 ID 参数。你可以使用一个未在顶层指定组件的路由来实现此目的。

在以下示例中，'MainChild' 和 'AuxChild' 是同级组件。当导航到 'parent/10/(a//aux:b)' 时，路由会实例化主子组件和辅助子组件，使它们彼此相邻。为了使此方法有效，应用组件必须定义主出口和辅助出口。

          
[{   path: 'parent/:id',   children: [     { path: 'a', component: MainChild },     { path: 'b', component: AuxChild, outlet: 'aux' }   ]}]

路由器将无组件父路由的参数、数据和求解合并到子路由的参数、数据和求解中。

当子组件使用空路径字符串定义时，这尤其有用，如以下示例所示。使用此配置，导航到 '/parent/10' 将创建主子组件和辅助组件。

          
[{   path: 'parent/:id',   children: [     { path: '', component: MainChild },     { path: '', component: AuxChild, outlet: 'aux' }   ]}]

惰性加载

惰性加载通过将应用拆分为多个捆绑包并按需加载它们来加快应用加载时间。要使用惰性加载，请在 Route 对象中提供 loadChildren 属性，而不是 children 属性。

给定以下示例路由，路由器将使用浏览器原生导入系统按需惰性加载关联的模块。

          
[{  path: 'lazy',  loadChildren: () => import('./lazy-route/lazy.module').then(mod => mod.LazyModule),}];
