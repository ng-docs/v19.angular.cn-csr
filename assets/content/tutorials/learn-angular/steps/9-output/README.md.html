使用 @Output 的组件通信

edit

在与组件一起工作时，可能需要通知其他组件发生了某些事情。 也许是按钮被点击了，条目已从列表中添加/删除，或者发生了一些其他重要的更新。 在这种情况下，组件需要与父组件通信。

Angular 使用 @Output 装饰器来启用这种类型的行为。

在此实践活动中，你将学习如何使用 @Output 装饰器和 EventEmitter 与组件通信。

要创建从子组件到父组件的通信路径，请在类属性上使用 @Output 装饰器，并为其赋予 EventEmitter 类型的值：

child.component.ts

      
@Component({...})class ChildComponent {    @Output() incrementCountEvent = new EventEmitter<number>();}

现在，该组件可以生成可由父组件监听的事件。 通过调用 emit 方法触发事件：

child.component.ts

      
class ChildComponent {    ...    onClick() {        this.count++;        this.incrementCountEvent.emit(this.count);    }}

emit 函数将生成一个与 EventEmitter 实例类型相同的事件。

好的，轮到你来尝试一下了。 通过完成以下任务来完成代码：

  1. 添加一个 @Output 属性

    更新 child.component.ts，添加一个名为 addItemEvent 的输出属性，请务必将 EventEmitter 类型设置为 string。

  2. 完成 addItem 方法

    在 child.component.ts 中，更新 addItem 方法； 使用以下代码作为逻辑：

    child.component.ts

          
    addItem() {  this.addItemEvent.emit('🐢');}

  3. 更新 AppComponent 模板

    在 app.component.ts 中，更新模板以监听发出的事件，方法是添加以下代码：

          
    <app-child (addItemEvent)="addItem($event)" />

    现在，“添加条目”按钮在每次单击时都会向列表中添加一个新条目。
