使用 RouterLink 进行导航

edit

在应用当前的状态下，当我们点击应用内部链接时，整个页面都会刷新。虽然这在小型应用中可能看起来并不重要，但对于内容较多的大型页面，这可能会对性能产生影响，因为用户必须重新下载资产并再次运行计算。

在本次实践中，你将学习如何利用 RouterLink 指令来充分利用 Angular Router。

  1. 导入 RouterLink 指令

    在 app.component.ts 中，将 RouterLink 指令导入添加到 @angular/router 中已有的导入语句中，并将其添加到组件装饰器的 imports 数组中。

          
    ...import { RouterLink, RouterOutlet } from '@angular/router';@Component({  imports: [RouterLink, RouterOutlet],  ...})

  2. 向模板添加 routerLink

    要使用 RouterLink 指令，请将 href 属性替换为 routerLink。使用此更改更新模板。

          
    import { RouterLink, RouterOutlet } from '@angular/router';@Component({  ...  template: `    ...    <a routerLink="/">Home</a>    <a routerLink="/user">User</a>    ...  `,  imports: [RouterLink, RouterOutlet],})

现在，当您点击导航中的链接时，您应该不会看到任何闪烁，并且只有页面本身的内容（即 router-outlet）正在更改 🎉

很好，路由相关的学习很棒。这只是 Router API 的冰山一角，要了解更多信息，请查看 Angular Router 文档。
